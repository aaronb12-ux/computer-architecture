## Finite State Machine Design Problem

### Steps:

1. **Write out transition table (without encodings).** Have a current state, input, and next state for rows and fill out this table based on the state diagram.

2. **Encode the states and outputs.** We have 3 states, so 2 bits of state are needed for each. S0 = 00, S1 = 01, S2 = 10, and our outputs are Q = 0 for S0/S1 and Q = 1 for S2.

3. **Create an encoded transition table** using these state encodings.

4. **Create next state equations** by finding the rows in S'₁ and S'₀ where the output is 1 and create SOP (Sum of Products) equations.

5. **Create output table.** For a Moore Model Machine, the output table has 2 columns: current state and output. We fill this out using the diagram and state encodings.

6. **Get output equation** using the SOP method from the output table.

7. **Draw the schematic** showing:
   - Combinational logic for next state
   - State registers (flip-flops)
   - Combinational logic for output

### Key Concepts:

**Moore Machine:** Output depends only on current state (not on input)

**State Encoding:**
- S0 = 00
- S1 = 01
- S2 = 10
- (11 unused)

**Next State Logic:** Determines S'₁S'₀ based on current state S₁S₀ and input A

**Output Logic:** Determines Q based on current state S₁S₀ only
