### a) Building a JK flip-flop from a D flip-flop

To create a JK flip-flop from a D flip-flop, we need to utilize a mux whose output feeds into the D port of the D flip-flop.

For this mux, it is a 4-to-1 mux because for a JK flip-flop, depending on J and K, there can be 4 outputs. The select bits are both J and K.

**Mux behavior:**
- If select bits are J = 0, K = 0, then send Q<sub>prev</sub> through (hold state)
- If J = 0, K = 1, then send 0 (reset)
- If J = 1, K = 0, then send 1 (set)
- If J = 1, K = 1, then send Q̄<sub>prev</sub> (toggle)

### b) Building a D flip-flop from a JK flip-flop

To build a D flip-flop from a JK flip-flop we utilize a mux approach also. We have two muxes which decide what J and K will be.

For these muxes our select bit is D.

**Mux behavior:**
- If D = 1, then through our mux logic, J = 1 and K = 0 (set)
- If D = 0, then J = 0 and K = 1 (reset)

This ensures the JK flip-flop behaves like a D flip-flop: Q = D on the next clock edge.

### c) Building a T flip-flop from a JK flip-flop

Same logic as above except invert the output Q, and feed it back into the select bit for both muxes.

**Alternative approach:** Simply tie J = 1 and K = 1 permanently. Since J = 1, K = 1 causes the JK flip-flop to toggle, this creates a T flip-flop behavior.

**Or using the mux approach:**
- Connect T to the select line of the muxes
- If T = 1, set J = 1 and K = 1 (toggle)
- If T = 0, set J = 0 and K = 0 (hold)s.
